✦ What's New

LangChain v0.3 重磅新特性

2024 年 9 月发布的 v0.3 是 LangChain 的一个重要里程碑，带来了 Pydantic 2 迁移、全新工具系统等关键升级。

📦

Pydantic 2 全面支持

所有包全面迁移至 Pydantic 2，提供更快的验证性能、更严格的类型检查，消除兼容桥接层。结构化输出更稳定可靠。

🔧

简化工具调用

全新的工具定义与调用 API，更直观的装饰器语法，自动参数注入，让 Function Calling 开发效率大幅提升。

🤖

增强的 Chat Models

统一的聊天模型接口，支持更丰富的消息类型（ToolMessage、AIMessageChunk），流式响应与结构化输出无缝集成。

🔗

LCEL 生产级就绪

LangChain Expression Language 支持流式、并行、Fallback 等生产场景，成为构建 Chain 的标准方式。

📊

LangGraph 为核心

Agent 底层全面基于 LangGraph 图运行时，带来持久化、回滚、Checkpoint、Human-in-the-Loop 等企业级能力。

🌐

集成生态扩展

全新 partner packages 架构（langchain-openai、langchain-anthropic、langchain-ollama 等），按需安装，依赖更轻量。

✦ Ecosystem

LangChain 生态全景

LangChain 已从单一框架演进为完整的 AI 工程平台，覆盖开发、编排、观测三大环节。

LangChain

核心框架 · v0.3+

提供 LLM 抽象、Prompt 模板、输出解析、RAG 管道、Agent 构建等基础组件。以模块化、可组合的方式构建 AI 应用，支持 100+ 模型与工具集成。

✦ Core Concepts

核心概念速览

理解这些基础概念，是掌握 LangChain 的关键一步。

01

Chat Models

LLM 交互的核心抽象。统一接口支持 OpenAI、Anthropic、Llama、Gemini 等主流模型，支持同步、流式、批量调用。

02

Prompt Templates

使用 ChatPromptTemplate 结构化构建 System / Human / AI 消息模板，支持变量插值与消息占位符，告别字符串拼接。

03

Output Parsers

将 LLM 的文本输出解析为结构化数据。支持 Pydantic 模型、JSON、XML、Datetime 等解析器，确保输出可控。

04

LCEL

LangChain Expression Language — 使用 | 操作符合成 Chain。天然支持流式、并行、Fallback 和异步执行。

05

RAG

检索增强生成。结合文档加载、文本分割、向量存储、检索与生成，让 LLM 基于私有知识回答问题。

06

Tools & Toolkits

@tool 装饰器快速定义工具，自动从函数签名生成 Schema。支持 LangChain Hub、API 工具、自定义工具。

07

Memory

对话状态管理。支持 Buffer、Summary、Vector Store 等多种记忆策略，让 Agent 记住上下文。

08

Agents

基于 LLM 的自主决策系统。结合 LangGraph 运行时，支持 ReAct、Plan-and-Execute 等模式，内置持久化与 HITL。

代码示例

新版 LangChain 的实际用法 — 点击标签切换示例。

# pip install langchain langchain-openai

from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

# v0.3+ 统一接口，支持所有主流模型
llm = ChatOpenAI(model="gpt-4o", temperature=0.7)

response = llm.invoke([
    HumanMessage(content="用中文解释 LangChain 是什么")
])
print(response.content)
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个{role}专家，用{language}回答。"),
    ("human", "{question}"),
])

chain = prompt | llm
result = chain.invoke({
    "role": "AI 架构",
    "language": "中文",
    "question": "什么是 RAG 技术？",
})
# LCEL — 使用 | 操作符合成 Chain
from langchain_core.output_parsers import StrOutputParser

chain = prompt | llm | StrOutputParser()

# 天然支持：流式输出
for chunk in chain.stream({"topic": "AI Agent"}):
    print(chunk, end="")

# 天然支持：Fallback
chain = chain.with_fallbacks([backup_chain])

# 天然支持：并行调用
from langchain_core.runnables import RunnableParallel
parallel = RunnableParallel({"qa": chain, "summary": summary_chain})
# RAG — 检索增强生成
from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import FAISS

# 1. 加载与分割
docs = TextLoader("doc.txt").load()
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)

# 2. 向量存储
vectorstore = FAISS.from_documents(chunks, OpenAIEmbeddings())
retriever = vectorstore.as_retriever()

# 3. RAG Chain
from langchain.chains import create_retrieval_chain
from langchain.chains.combine_documents import create_stuff_documents_chain

rag_chain = create_retrieval_chain(retriever, create_stuff_documents_chain(llm, prompt))
result = rag_chain.invoke({"input": "文档的主要内容是什么？"})
# v0.3+ 简化工具定义
from langchain_core.tools import tool

@tool
def get_weather(city: str) -> str:
    "查询指定城市的天气"
    return f"{city}今天晴朗，22°C"

tools = [get_weather]

# Agent 构建（基于 LangGraph）
from langchain.agents import create_tool_calling_agent
from langchain.agents import AgentExecutor

agent = create_tool_calling_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools)

result = agent_executor.invoke({"input": "北京和上海的天气分别是？"})
print(result["output"])
