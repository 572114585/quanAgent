01 · LangChain 是什么

一个面向 LLM 应用开发的综合性框架，从原型到生产全覆盖

🧩 框架定位

LangChain 是一个用于构建 LLM 驱动应用的开发框架，由 Harrison Chase 于 2022 年创建，目前是 AI 应用开发领域最受欢迎的开源项目之一（GitHub 90k+ Stars）。

2025 年 10 月发布的 v1.0 是首个稳定主版本，标志着框架从快速迭代进入成熟期。核心变化包括：统一的 create_agent API、全新的中间件系统、以及围绕 LangGraph 运行时的重构。

🏗️ 核心架构分层

LangChain v1.0 采用清晰的分层架构，各层职责明确：

⬇

Retrieval

💡

v1.0 核心变化

Agent 创建统一为 create_agent 函数（替代 create_react_agent），引入中间件机制实现可插拔的行为拦截，结构化输出成为一等公民。

📜 版本演进

2022 · 诞生

LangChain 0.0.x

Harrison Chase 创建项目，开创性地提出 "Chain" 概念和 LLM 应用组合范式。

2023 · 爆发

LangChain 0.1 - 0.2

社区爆炸式增长，引入 LCEL、Tool Calling、Agent 等核心能力。

2024 · 成熟

LangChain 0.3

模块化重构，拆分 langchain-core、langchain-community，增强多模态支持。

2025 · 里程碑

LangChain 1.0 · LangGraph 1.0

首个主版本！统一的 Agent API、中间件架构、结构化输出、生产级稳定性承诺。

02 · 快速开始

5 分钟搭建你的第一个 LangChain 应用

📦 安装

LangChain v1.0 支持 Python 3.9+，推荐使用 pip 安装：

bash
# 安装核心包
pip install langchain langchain-core langchain-community

# 安装模型提供商（根据需求选择）
pip install langchain-openai   # OpenAI / Azure OpenAI
pip install langchain-anthropic # Anthropic Claude
pip install langchain-google    # Google Gemini

# 安装 LangGraph（编排引擎）
pip install langgraph

# 安装向量存储
pip install langchain-chroma    # Chroma
pip install langchain-pinecone  # Pinecone

👋 第一个 LangChain 程序

Python
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)
messages = [
    SystemMessage(content="你是一个专业的 Python 技术顾问。"),
    HumanMessage(content="请用 Python 实现一个快速排序算法"),
]
response = llm.invoke(messages)
print(response.content)
💡

环境变量

确保已设置 OPENAI_API_KEY 环境变量，或在初始化时通过 api_key 参数传入。

🤖 第一个 Agent

Python
from langchain_openai import ChatOpenAI
from langchain.agents import create_agent, tool
from langchain_core.messages import HumanMessage

@tool
def calculate(expression: str) -> float:
    """计算数学表达式的结果"""
    return eval(expression)

agent = create_agent(
    model=ChatOpenAI(model="gpt-4o-mini"), tools=[calculate],
)
result = agent.invoke({
    "messages": [HumanMessage(content="计算 (23 × 45) + 178 的结果")]
})
print(result["messages"][-1].content)

04 · RAG 检索增强生成

让 LLM 拥有外部知识库，突破上下文窗口限制

📡 RAG 全流程

RAG 流程：文档加载 → 文本分割 → 向量存储 → 检索 → 生成

文档加载
→
文本分割
→
向量存储
→
检索
→
生成
Python
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain_core.runnables import RunnablePassthrough

loader = PyPDFLoader("document.pdf")
docs = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(docs)

vectorstore = Chroma.from_documents(chunks, OpenAIEmbeddings(model="text-embedding-3-small"))
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

prompt = ChatPromptTemplate.from_messages([
    ("system", "基于上下文回答问题：\n\n{context}"),
    ("human", "{question}"),
])

def format_docs(docs):
    return "\n\n".join(d.page_content for d in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt | llm
)
result = rag_chain.invoke("文档中提到的核心算法是什么？")

🗂️ 文档加载器 & 文本分割

分割器策略最佳场景
RecursiveCharacterTextSplitter递归按分隔符切割通用文本
TokenTextSplitter按 token 数切割精确控制上下文
SemanticChunker基于语义边界切割最佳检索质量

文档加载器支持：📄 PDF · 🌐 网页 · 💻 代码 · 📊 CSV/JSON · 🔊 音视频 · 🏛️ 数据库

05 · Chains & Runnable 接口

LCEL 表达式语言，用管道符组合复杂 AI 逻辑

⛓️ LCEL 基础

一切皆为 Runnable，通过 | 管道操作符组合：

Python
from langchain_core.runnables import RunnablePassthrough, RunnableParallel, RunnableBranch

# 链式调用
chain = prompt_template | llm | output_parser
result = chain.invoke({"topic": "Python 异步"})

# 并行执行
parallel = RunnableParallel({
    "summary": summary_chain, "keywords": keywords_chain,
})
results = parallel.batch([{"text": "你好"}])

# 流式输出
for chunk in chain.stream({"topic": "ML"}):
    print(chunk, end="")

# 条件路由
branch = RunnableBranch(
    (lambda x: "技术" in x["topic"], tech_chain),
    default_chain,
)

06 · 记忆系统

让 LLM 应用拥有持续对话和多轮上下文理解能力

🧠 记忆类型与使用

记忆类型工作机制适用场景
ConversationBufferMemory完整保留所有对话短对话
ConversationSummaryMemoryLLM 自动总结长对话，省 token
ConversationBufferWindowMemory保留最近 k 轮平衡与消耗
VectorStoreRetrieverMemory向量检索历史超长对话
Python
from langchain.memory import ConversationSummaryMemory
from langchain.chains import LLMChain

memory = ConversationSummaryMemory(
    llm=ChatOpenAI(model="gpt-4o-mini"),
    memory_key="chat_history", return_messages=True,
)
chain = LLMChain(llm=llm, prompt=prompt, memory=memory)
chain.invoke({"input": "你好，我叫小明"})
chain.invoke({"input": "你还记得我叫什么吗？"})  # → "小明"
⚠️

生产环境注意

内置记忆适用于原型。生产推荐 LangGraph 状态管理 + Redis/PostgreSQL 持久化。

07 · Agent 智能体

v1.0 核心 — 统一的 create_agent API，工具调用、中间件、结构化输出

🤖 create_agent — 新一代 API

Python
from langchain.agents import create_agent, tool
from langchain_core.messages import HumanMessage

@tool
def get_weather(city: str) -> str:
    """查询指定城市的当前天气"""
    return f"{city} 当前气温 22°C，多云"

@tool
def search_web(query: str) -> str:
    """搜索网络信息"""
    return f"搜索结果: {query}"

agent = create_agent(
    model=ChatOpenAI(model="gpt-4o"),
    tools=[get_weather, search_web],
    prompt="你是一个有用的助手，请使用工具回答问题。",
)
result = agent.invoke({
    "messages": [HumanMessage(content="北京的天气怎么样？顺便搜一下今日热点。")]
})
for msg in result["messages"]:
    print(f"[{msg.type}] {msg.content}")

🔌 中间件系统

Python
from langchain.agents.middleware import Middleware

class LoggingMiddleware(Middleware):
    async def __call__(self, params, next):
        print(f"[调用前] 模型: {params.model}")
        result = await next(params)
        print(f"[调用后] Token: {result.usage}")
        return result

agent = create_agent(model=llm, tools=[...],
    middleware=[LoggingMiddleware()])
# 内置: ModelRetryMiddleware, HumanInTheLoopMiddleware, RateLimitMiddleware

08 · LangGraph 工作流

图状态机引擎 — 构建复杂、可靠的多步 Agent 工作流

🕸️ 核心概念

LangGraph 以有向图定义执行流程，基于状态机设计：

State
应用状态
→
Node
处理逻辑
→
Edge
条件路由
→
Graph
完整工作流
Python
from typing import TypedDict, List
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver

class AgentState(TypedDict):
    messages: List
    next_step: str

def classify(state): return {"next_step": "generate"}
def generate(state): return state

builder = StateGraph(AgentState)
builder.add_node("classify", classify)
builder.add_node("generate", generate)
builder.add_edge(START, "classify")
builder.add_conditional_edges("classify", lambda s: s["next_step"])
builder.add_edge("generate", END)

graph = builder.compile(checkpointer=MemorySaver())

🔁 循环 Agent 与持久化

Python
# 循环 Agent 工作流
from langgraph.prebuilt import tools_condition

builder = StateGraph(AgentState)
builder.add_node("agent", call_model)
builder.add_node("tools", tool_executor)
builder.add_edge(START, "agent")
builder.add_conditional_edges("agent", tools_condition)
builder.add_edge("tools", "agent")

# 持久化
from langgraph.checkpoint.sqlite import SqliteSaver
graph = builder.compile(checkpointer=SqliteSaver(conn))

🚨 常见陷阱

⚠️

Token 消耗失控

设置 max_turns 限制，使用摘要记忆压缩历史。

⚠️

工具调用失败

始终为工具添加错误处理，利用中间件实现重试和降级。

⚠️

状态管理混乱

使用 TypedDict + Checkpointer 确保状态可追踪。
